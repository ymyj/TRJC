# 管理类模块开发指南

本文档详细说明如何在 TRJC 系统中快速构建类似"人员管理"的通用管理模块（增删改查），包括后端和前端的完整开发流程。可作为后续开发的经验参考。

---

## 一、后端开发流程

### 第1步：定义数据库模型（Model）

**文件位置**: `TRJC-backend/app/models/__init__.py`

**技术要求**:
- 继承 `Base` 类（SQLAlchemy ORM）
- 使用 `__tablename__` 定义表名（命名规范：模块名_info，如 `person_info`）
- 必须包含的标准字段：
  - `ID`: `Integer, primary_key=True, autoincrement=True` - 主键
  - `CJSJ`: `DateTime, server_default=func.now()` - 创建时间
  - `SFSC`: `TINYINT, default=0` - 逻辑删除标记（0未删除，1已删除）
- 业务字段根据需求定义，敏感数据（姓名、电话、密码）使用 `String(200)` 准备加密存储
- 使用 `Column()` 定义字段，添加 `comment` 注释说明

**示例代码**:
```python
class XxxInfo(Base):
    __tablename__ = "xxx_info"
    
    ID = Column(Integer, primary_key=True, autoincrement=True, comment="主键")
    # ... 业务字段
    CJSJ = Column(DateTime, server_default=func.now(), comment="创建时间")
    SFSC = Column(TINYINT, default=0, comment="是否删除：0-未删除，1-已删除")
```

### 第2步：定义 Pydantic Schema（数据验证）

**文件位置**: `TRJC-backend/app/schemas/xxx.py`

**技术要求**:
- 创建 4-5 个 Schema 类：
  - `XxxBase`: 基础字段（所有必填字段）
  - `XxxCreate`: 继承 `Base`（用于新增操作）
  - `XxxUpdate`: 所有字段 `Optional`（用于更新操作）
  - `XxxResponse`: 完整返回结构（包含 ID、CJSJ 等）
  - `XxxListResponse`: 列表返回结构（可省略部分字段）
- 所有 Response 类必须设置：`class Config: from_attributes = True`
- 使用 `typing.Optional` 标记可选字段

**示例代码**:
```python
from pydantic import BaseModel
from typing import Optional
from datetime import datetime

class XxxBase(BaseModel):
    name: str
    status: str = "active"

class XxxCreate(XxxBase):
    pass

class XxxUpdate(BaseModel):
    name: Optional[str] = None
    status: Optional[str] = None

class XxxResponse(BaseModel):
    ID: int
    name: str
    status: str
    CJSJ: Optional[datetime] = None
    SFSC: int = 0

    class Config:
        from_attributes = True
```

### 第3步：实现 API 路由和控制器

**文件位置**: `TRJC-backend/app/api/xxx.py`

**标准接口清单**:

| 方法 | 路径 | 说明 | 参数 |
|------|------|------|------|
| GET | `/api/xxx` | 获取列表（分页+搜索） | page, size, keyword, 过滤字段 |
| GET | `/api/xxx/options` | 获取下拉选项 | - |
| GET | `/api/xxx/{id}` | 获取详情 | id |
| POST | `/api/xxx` | 新增数据 | JSON Body |
| PUT | `/api/xxx/{id}` | 更新数据 | id, JSON Body |
| DELETE | `/api/xxx/{id}` | 删除数据（逻辑删除） | id |

**技术要求**:
- 使用 `APIRouter(prefix="/api/xxx", tags=["模块名称"])`
- 所有接口返回格式统一：`{"code": 200, "msg": "", "data": {}}`
- 列表接口支持分页参数：`page`, `size`（使用 `Query` 验证）
- 列表接口支持搜索参数：`keyword`, 业务过滤字段
- **必须过滤** `SFSC == 0` 的数据（逻辑删除）
- 敏感数据加密：使用 `encrypt_data()`/`decrypt_data()`
- 错误处理：使用 `HTTPException(status_code=404, detail="")`
- 数据库操作：`db: Session = Depends(get_db)`

**示例代码（列表接口）**:
```python
from fastapi import APIRouter, Depends, HTTPException, Query
from sqlalchemy.orm import Session
from typing import Optional
from app.database import get_db
from app.models import XxxInfo
from app.schemas.xxx import XxxCreate, XxxUpdate

router = APIRouter(prefix="/api/xxx", tags=["模块名"])

@router.get("", response_model=dict)
def get_xxx_list(
    page: int = Query(1, ge=1),
    size: int = Query(10, ge=1, le=100),
    keyword: Optional[str] = None,
    db: Session = Depends(get_db)
):
    query = db.query(XxxInfo).filter(XxxInfo.SFSC == 0)
    
    if keyword:
        query = query.filter(XxxInfo.name.like(f"%{keyword}%"))
    
    total = query.count()
    items = query.offset((page - 1) * size).limit(size).all()
    
    result = [
        {
            "ID": item.ID,
            "name": item.name,
            "status": item.status,
            "CJSJ": item.CJSJ
        }
        for item in items
    ]
    
    return {"code": 200, "data": {"list": result, "total": total, "page": page, "size": size}}

@router.post("", response_model=dict)
def create_xxx(data: XxxCreate, db: Session = Depends(get_db)):
    db_item = XxxInfo(
        name=data.name,
        status=data.status
    )
    db.add(db_item)
    db.commit()
    db.refresh(db_item)
    return {"code": 200, "msg": "添加成功", "data": {"ID": db_item.ID}}

@router.put("/{xxx_id}", response_model=dict)
def update_xxx(xxx_id: int, data: XxxUpdate, db: Session = Depends(get_db)):
    item = db.query(XxxInfo).filter(XxxInfo.ID == xxx_id, XxxInfo.SFSC == 0).first()
    if not item:
        raise HTTPException(status_code=404, detail="数据不存在")
    
    if data.name is not None:
        item.name = data.name
    if data.status is not None:
        item.status = data.status
    
    db.commit()
    return {"code": 200, "msg": "更新成功"}

@router.delete("/{xxx_id}", response_model=dict)
def delete_xxx(xxx_id: int, db: Session = Depends(get_db)):
    item = db.query(XxxInfo).filter(XxxInfo.ID == xxx_id, XxxInfo.SFSC == 0).first()
    if not item:
        raise HTTPException(status_code=404, detail="数据不存在")
    
    item.SFSC = 1
    db.commit()
    return {"code": 200, "msg": "删除成功"}
```

### 第4步：注册路由

**文件位置**: `TRJC-backend/app/main.py`

**操作步骤**:
1. 在 import 语句中添加：`from app.api import xxx`
2. 在 `app.include_router()` 部分添加：`app.include_router(xxx.router)`
3. 保持路由注册顺序合理（auth 优先）

**示例**:
```python
from app.api import auth, personnel, plots, tasks, xxx

app.include_router(auth.router)
app.include_router(personnel.router)
# ... 其他路由
app.include_router(xxx.router)
```

### 第5步：数据库表创建

**说明**:
- 当前系统使用 `Base.metadata.create_all(bind=engine)` 自动建表
- 在 `main.py` 启动时自动创建所有定义的模型表
- 重启后端服务即可自动创建新表

**注意**: 生产环境建议使用专业的迁移工具如 Alembic

## 二、前端开发流程

### 第1步：创建 API 接口文件

**文件位置**: `TRJC-web/src/api/xxx.js`

**技术要求**:
- 使用已封装的 `api` 实例（来自 `./request`）
- 导出所有 CRUD 函数，函数命名规范：
  - `getXxxList(params)`
  - `getXxxOptions()`
  - `getXxxDetail(id)`
  - `createXxx(data)`
  - `updateXxx(id, data)`
  - `deleteXxx(id)`
- 使用对应的 HTTP 方法：`api.get`/`api.post`/`api.put`/`api.delete`
- 路径与后端路由完全对应

**示例代码**:
```javascript
import api from './request'

export function getXxxList(params) {
  return api.get('/api/xxx', { params })
}

export function getXxxOptions() {
  return api.get('/api/xxx/options')
}

export function createXxx(data) {
  return api.post('/api/xxx', data)
}

export function updateXxx(id, data) {
  return api.put(`/api/xxx/${id}`, data)
}

export function deleteXxx(id) {
  return api.delete(`/api/xxx/${id}`)
}

export function getXxxDetail(id) {
  return api.get(`/api/xxx/${id}`)
}
```

### 第2步：创建 Vue 页面组件

**文件位置**: `TRJC-web/src/views/XxxManagement.vue`

**页面结构要求**:

1. **筛选区域**（`filter-section`）
   - 输入框、下拉选择器等筛选条件
   - 查询、重置、新增按钮

2. **数据表格区域**（`list-section`）
   - 表头（列名与业务字段对应）
   - 数据行（`v-for` 遍历列表）
   - 操作列（编辑、删除按钮）
   - 空数据提示

3. **分页组件**（`pagination-wrapper`）
   - 记录数统计
   - 页码按钮

4. **新增/编辑弹窗**（`modal-overlay`）
   - 表单区域（`form-row` + `form-item`）
   - 取消、确定按钮

**技术要求**:
- 使用 Composition API（`<script setup>`）
- 使用 `ref` 和 `reactive` 管理状态
- 数据请求在 `onMounted` 中初始化
- 响应格式：`res.data.code === 200` 判断成功
- 统一使用现有 CSS 类名保持风格一致

**核心逻辑示例**:
```vue
<script setup>
import { ref, reactive, onMounted } from 'vue'
import { getXxxList, createXxx, updateXxx, deleteXxx } from '../api/xxx'

// 筛选表单
const filterForm = reactive({
  keyword: '',
  status: ''
})

// 数据列表
const xxxList = ref([])

// 分页配置
const pagination = reactive({
  current: 1,
  total: 0,
  pageSize: 10,
  get totalPages() { return Math.ceil(this.total / this.pageSize) }
})

// 弹窗控制
const showAddModal = ref(false)
const isEdit = ref(false)
const addForm = reactive({
  name: '',
  status: 'active'
})

// 获取列表
const fetchList = async () => {
  try {
    const params = {
      page: pagination.current,
      size: pagination.pageSize,
      keyword: filterForm.keyword || undefined,
      status: filterForm.status || undefined
    }
    const res = await getXxxList(params)
    if (res.data.code === 200) {
      xxxList.value = res.data.data.list || []
      pagination.total = res.data.data.total || 0
    }
  } catch (error) {
    console.error('获取列表失败:', error)
  }
}

// 查询
const handleQuery = () => {
  pagination.current = 1
  fetchList()
}

// 重置
const resetFilter = () => {
  filterForm.keyword = ''
  filterForm.status = ''
  pagination.current = 1
  fetchList()
}

// 新增
const handleAdd = () => {
  isEdit.value = false
  showAddModal.value = true
  Object.assign(addForm, { name: '', status: 'active' })
}

// 编辑
const handleEdit = (item) => {
  isEdit.value = true
  showAddModal.value = true
  Object.assign(addForm, { ...item })
}

// 确认提交
const confirmAdd = async () => {
  if (!addForm.name) {
    alert('请填写必填项')
    return
  }
  try {
    if (isEdit.value) {
      await updateXxx(addForm.ID, addForm)
      alert('更新成功')
    } else {
      await createXxx(addForm)
      alert('添加成功')
    }
    closeAddModal()
    fetchList()
  } catch (error) {
    console.error('操作失败:', error)
    alert('操作失败')
  }
}

// 关闭弹窗
const closeAddModal = () => {
  showAddModal.value = false
  Object.assign(addForm, { name: '', status: 'active' })
}

// 删除
const handleDelete = async (item) => {
  if (!confirm(`确定删除该记录吗？`)) return
  try {
    await deleteXxx(item.ID)
    alert('删除成功')
    fetchList()
  } catch (error) {
    console.error('删除失败:', error)
    alert('删除失败')
  }
}

// 切换页码
const changePage = (page) => {
  if (page >= 1 && page <= pagination.totalPages) {
    pagination.current = page
    fetchList()
  }
}

onMounted(() => {
  fetchList()
})
</script>
```

## 三、前后端对接关键约定

### 1. 数据格式约定

**后端统一返回格式**:
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```

**列表接口返回格式**:
```json
{
  "code": 200,
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "size": 10
  }
}
```

**前端判断成功**:
```javascript
if (res.data.code === 200) {
  // 处理成功逻辑
}
```

### 2. 敏感数据处理

- **后端存储**: 使用 `encrypt_data()` 加密（姓名、电话、密码等）
- **后端返回**: 使用 `decrypt_data()` 解密，列表接口使用 `mask_name()`/`mask_phone()` 脱敏
- **前端**: 正常显示，无需额外处理

**示例**:
```python
# 新增时加密
db_item = PersonInfo(
    XM=encrypt_data(data.XM),
    LXFS=encrypt_data(data.LXFS)
)

# 列表返回时脱敏
result.append({
    "XM": mask_name(decrypt_data(item.XM)),
    "LXFS": mask_phone(decrypt_data(item.LXFS))
})

# 详情返回时解密
return {
    "XM": decrypt_data(item.XM),
    "LXFS": decrypt_data(item.LXFS)
}
```

### 3. 逻辑删除机制

- 删除操作**不物理删除**数据，而是设置 `SFSC = 1`
- 所有查询必须添加过滤条件：`filter(XxxInfo.SFSC == 0)`
- 前端删除按钮需二次确认

### 4. 分页参数传递

**前端 → 后端**:
```javascript
{
  page: 1,        // 当前页码（从1开始）
  size: 10,       // 每页条数
  keyword: "",    // 搜索关键词
  status: ""      // 业务过滤字段
}
```

**后端 → 前端**:
```javascript
{
  list: [],       // 当前页数据
  total: 100,     // 总记录数
  page: 1,        // 当前页码
  size: 10        // 每页条数
}
```

### 5. 表单提交流程

**新增流程**:
1. 前端校验必填项
2. 调用 `createXxx(addForm)`
3. 成功后关闭弹窗，刷新列表

**编辑流程**:
1. 打开弹窗时填充表单：`Object.assign(form, data)`
2. 调用 `updateXxx(id, updateForm)`
3. 成功后关闭弹窗，刷新列表

**删除流程**:
1. 二次确认
2. 调用 `deleteXxx(id)`
3. 成功后刷新列表

## 五、常见问题

### Q1: 如何添加新的筛选条件？

**后端**: 在列表接口添加 Query 参数并增加过滤逻辑
```python
@router.get("", response_model=dict)
def get_xxx_list(
    page: int = Query(1, ge=1),
    size: int = Query(10, ge=1),
    status: Optional[str] = None,  # 新增过滤参数
    db: Session = Depends(get_db)
):
    query = db.query(XxxInfo).filter(XxxInfo.SFSC == 0)
    if status:
        query = query.filter(XxxInfo.status == status)
```

**前端**: 在 filterForm 添加字段并在 fetchList 中传递
```javascript
const filterForm = reactive({ keyword: '', status: '' })

const fetchList = async () => {
  const params = {
    page: pagination.current,
    size: pagination.pageSize,
    status: filterForm.status || undefined
  }
  const res = await getXxxList(params)
}
```

### Q2: 如何实现批量删除？

**后端**: 添加批量删除接口
```python
@router.post("/batch-delete", response_model=dict)
def batch_delete_xxx(ids: list[int], db: Session = Depends(get_db)):
    items = db.query(XxxInfo).filter(XxxInfo.ID.in_(ids), XxxInfo.SFSC == 0).all()
    for item in items:
        item.SFSC = 1
    db.commit()
    return {"code": 200, "msg": f"成功删除{len(items)}条记录"}
```

**前端**: 添加批量选择和批量删除逻辑
```javascript
const selectedIds = ref([])

const handleBatchDelete = async () => {
  if (selectedIds.value.length === 0) return
  if (!confirm(`确定删除选中的${selectedIds.value.length}条记录吗？`)) return
  try {
    await batchDeleteXxx(selectedIds.value)
    alert('删除成功')
    fetchList()
  } catch (error) {
    alert('删除失败')
  }
}
```

### Q3: 如何处理关联数据？

对于有关联关系的模块（如任务关联人员、地块），建议：
1. 使用关联表存储关系（如 `task_assign` 表）
2. 提供 options 接口获取可选数据
3. 前端使用下拉选择器关联

### Q4: 如何实现导出功能？

**后端**: 添加导出接口，返回 CSV/Excel 格式
```python
from fastapi.responses import Response
import csv
import io

@router.get("/export", response_model=dict)
def export_xxx(
    keyword: Optional[str] = None,
    db: Session = Depends(get_db)
):
    query = db.query(XxxInfo).filter(XxxInfo.SFSC == 0)
    if keyword:
        query = query.filter(XxxInfo.name.like(f"%{keyword}%"))
    
    items = query.all()
    
    # 生成 CSV
    output = io.StringIO()
    writer = csv.writer(output)
    writer.writerow(['ID', '姓名', '状态', '创建时间'])
    for item in items:
        writer.writerow([item.ID, item.name, item.status, item.CJSJ])
    
    return Response(
        content=output.getvalue(),
        media_type="text/csv",
        headers={"Content-Disposition": "attachment; filename=xxx_export.csv"}
    )
```

### Q5: 如何实现导入功能？

**前端**: 使用文件上传组件
**后端**: 解析文件并批量导入
```python
from fastapi import UploadFile, File
import csv

@router.post("/import", response_model=dict)
def import_xxx(file: UploadFile = File(...), db: Session = Depends(get_db)):
    content = file.file.read().decode('utf-8')
    reader = csv.reader(io.StringIO(content))
    next(reader)  # 跳过表头
    
    count = 0
    for row in reader:
        db_item = XxxInfo(
            name=row[0],
            status=row[1]
        )
        db.add(db_item)
        count += 1
    
    db.commit()
    return {"code": 200, "msg": f"成功导入{count}条记录"}
```
